# Architecture Overview

This repository now centres on a reproducible pipeline for benchmarking brain-tumour segmentation CNNs. The flow is organised into four layers: data generation, model registry, training/evaluation, and reporting.

## 1. Data Generation & Caching
- **Module**: data_processing.py, dataset_refactored.py
- Synthetic MRI volumes and segmentation masks are produced by BrainStructureGenerator, which blends multiple rotated Gaussian lobes and attention to produce realistic tumours (necrotic core, enhancing rim, diffuse oedema).
- BrainTumorDataset handles preprocessing and caching. Generated samples (image, mask, underlying structures) are saved into dataset_cache/ and automatically re-used unless egenerate_cache=True in the dataset config.
- create_dataloaders offers train/val/test loaders with shared cache configuration so every training run consumes the same data splits.

## 2. Model Registry
- **Package**: models/
- models/registry.py exposes egister_model, get_model, list_models. Each architecture self-registers by importing egister_model and providing metadata/default kwargs.
- Implementations live in models/basic.py (Basic & Standard U-Net variants) and models/advanced.py (attention-enhanced residual U-Net). Legacy files (asic_unet_refactored.py, dvanced_brain_tumor_unet.py) are thin shims to maintain backwards compatibility.

## 3. Training & Evaluation Infrastructure
- **Package**: 	raining/
  - 	raining/engine.py houses TrainingEngine (per-epoch orchestration) and TrainingRunMetrics. It now tracks confusion matrices alongside Dice/IoU.
  - 	raining/metrics.py centralises Dice/IoU/accuracy helpers, multi-class metrics aggregation, and confusion-matrix utilities.
  - 	raining/runner.py adds TrainEvalRunner, which coordinates optimisers (AdamW), learning-rate scheduling (ReduceLROnPlateau), gradient clipping, checkpointing, and early stopping. The runner returns a TrainingSummary containing history, best validation metrics, checkpoints, and aggregate confusion.

## 4. Pipeline Orchestrator
- **Package**: pipeline/
  - pipeline/config.py bundles dataclasses describing the dataset, per-model training overrides, and reporting options. PipelineConfig.default() defines sensible defaults (3 models, cached synthetic data, shared loss).
  - pipeline/run_pipeline.py exposes execute_pipeline(cfg) plus a CLI entry point (python -m pipeline.run_pipeline --config optional.json). The executor:
    1. Loads/merges JSON overrides into a PipelineConfig.
    2. Builds dataloaders via dataset_refactored.create_dataloaders.
    3. Iterates registered models, instantiates each through the registry, constructs the runner, and executes the training loop.
    4. Collects ModelPipelineResult objects (summaries, metrics, parameter counts).

## 5. Reporting
- **Package**: eporting/
  - eporting/generator.py converts pipeline results into CSV/JSON summaries, Markdown overviews, per-model training history plots, and confusion-matrix heatmaps. Outputs live under eports/ by default.
  - eporting/__init__.py exports generate_reports for reuse.

## 6. Legacy Entrypoints
- comprehensive_model_evaluation.py now delegates to the pipeline with larger dataset/sample configuration.
- Other legacy scripts (quick_comprehensive_evaluation.py, main.py, etc.) remain for reference but are superseded by the pipeline.

## 7. Extending the System
1. **Add a new model**: implement the architecture in models/your_model.py, import egister_model, and call egister_model("your_model", constructor, metadata).
2. **Adjust training**: update PipelineConfig.default() or provide a JSON config overriding optimiser, scheduler, epochs, loss parameters, or dataset sizes.
3. **Additional metrics/reporting**: extend 	raining/metrics.py to compute new statistics and update eporting/generator.py to visualise or log them.
4. **Integrate real datasets**: create a new Dataset subclass and wire it into create_dataloaders or build a complementary factory.

## 8. Execution
`ash
# Quick run with defaults (auto CPU fallback)
python -m pipeline.run_pipeline

# Run using a custom config (JSON)
python -m pipeline.run_pipeline --config configs/experiment.json

# Comprehensive evaluation wrapper (uses larger splits)
python comprehensive_model_evaluation.py
`

Reports (CSV, JSON, Markdown, figures) are written to the configured eports/ directory. Cached synthetic data sits in dataset_cache/ and can be regenerated by toggling egenerate_cache in the dataset config.
